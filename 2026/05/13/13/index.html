PenGym 单智能体 izumi 当日进展报告：small-family 完成 clean demo + BC + actor-freeze，并通过 2-expert router 达到 40/40 全覆盖

一、这篇文章要说明什么

上一篇记录结束时，项目已经完成了一个比较关键的 family-level 结果：

  • 当前主线固定为 646/192，即 obs_dim=646、action_dim=192；
  • tiny-family 不再依赖单一 BC policy，而是通过 2-expert router 实现全覆盖；
  • tiny 使用 maskable_ppo_izumi_tiny_adapt_20k.zip；
  • tiny-small / tiny-hard 共用 tiny-small successdistill actor-freeze PPO expert；
  • tiny / tiny-small / tiny-hard 三个场景最终达到 deterministic 60/60 成功；
  • 这说明当前研究已经从“单场景策略训练”推进到“solved experts + router”的 family-level routing 阶段。

因此，本阶段的重点从 tiny-family 转向下一层：

从 tiny-family solved 推进到 small-family solved，并为后续 medium / small / tiny cross-family router 做准备。

本阶段的最终结果可以概括为十四点：

  1. 已扫描并确认当前 small family 主要包含 small-honeypot.yml 与 small-linear.yml 两个场景。
  2. 已发现仓库中存在旧的 small-honeypot adaptation 模型，但 full deterministic eval 失败。
  3. 已确认旧 small 模型本身是 646/192，wrapper stack 与当前主线兼容；失败不是维度不兼容导致。
  4. 已重新训练 small-honeypot 50k PPO adaptation，但 deterministic eval 仍为 0/40，说明继续直接 PPO 堆步数不是优先路线。
  5. 已用 50k 模型进行 stochastic success trace collection，并成功收集到 2 条 small-honeypot 成功轨迹。
  6. 已尝试直接用 stochastic success traces 训练 BC，但 deterministic eval 失败，并出现过早执行终结 exploit 的问题。
  7. 已通过 logits 诊断确认：small-honeypot 的终结动作不是 e_ftp@(4,0)，而是 pe_tomcat@(4,0)；BC 失败的原因不是不会选终结动作，而是太早选了它。
  8. 已尝试 guarded BC eval，成功打断 pe_tomcat@(4,0) 的 connection-error 死循环，但策略转入新的 7 动作重复环。
  9. 已转向人工 clean demo 路线，并成功构建 small-honeypot clean unified dataset，训练出 deterministic 20/20 的 clean BC。
  10. 已进一步通过 actor-freeze / value-only PPO 得到 small-honeypot solved expert，但该 expert zero-shot 到 small-linear 失败。
  11. 已修复 small-linear clean demo 中的动作名映射问题，将不被 builder 支持的 PrivilegeEscalation_Daclsvc 修正为 PrivilegeEscalation_Proftpd。
  12. 已成功构建 small-linear clean unified dataset，并训练出 deterministic 20/20 的 small-linear clean BC。
  13. 已进一步通过 actor-freeze / value-only PPO 得到 small-linear solved expert。
  14. 已新增 small-family router，将 small-honeypot 与 small-linear 分别路由到各自 solved expert，最终达到 40/40 成功。

本阶段最重要的结论是：

small-family 已经通过 clean manual demo → unified dataset → BC → actor-freeze PPO 的路线分别解决 small-honeypot 与 small-linear，并最终通过 2-expert router 达到 deterministic 40/40 全覆盖。

这意味着第 13 阶段已经从“small-honeypot 单点 solved”推进为真正的：

十八、当前阶段的关键技术结论

1. 旧 small-honeypot PPO 资产不可直接复用

虽然旧模型是 646/192 且 wrapper 兼容，但 full deterministic eval 失败。因此它不能作为 solved expert。

2. 继续直接 PPO adaptation 没有解决 small-honeypot

20k 与 50k 都没有达到 deterministic solved，说明第 13 阶段不应继续盲目堆 PPO timesteps。

3. stochastic success traces 有价值，但不能直接蒸馏成 BC solved

两条 stochastic success traces 能证明策略空间里存在成功路径，但 trace 太乱，直接 BC 会学到循环动作。

4. small-honeypot 的终结动作口径必须修正
